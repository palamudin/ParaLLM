    Settings / fine tuning

    Runtime controls

    Budgets, model choices, loop depth, and research policy live here.
    Profiles retune models, reasoning effort, auto-loop depth, and spend ceilings without making people hand-tune every lane.
    Profiles change worker default model, summarizer model, reasoning effort, auto-loop depth, and session spend wall. Research and vetting still stay manual.
    `Full` keeps the standard single conversation view. `Eval` runs the pressurized answer and the single-thread baseline through the front together, then asks an external arbiter to score the finished pair.
    `Light Workers` keeps full context on the main thread, while adversarial lanes receive weighted digests built from the commander draft, prior summary, peer steer, and current constraints. `Full Workers` sends the broader packet to workers too.
    `Off` keeps the current pressurized ParaLLM path only. `Single Only` runs one direct answer pipe without worker pressure. `Both Compare` runs the direct answer once alongside the pressurized loop so Review can compare them.
    Only used when the single-thread baseline is enabled.
    Used when workers or the summarizer run on Ollama. Enter the host URL like `http://192.168.0.26:11434` or an `/api` base URL. ParaLLM will route live Ollama calls there instead of assuming localhost.
    Only used when an Ollama provider is active.
    Profiles are the fast path. These knobs are still here when you want to fine-tune the current template before the next send.
    Commander and workers can inspect read-only local files only inside these repo-relative roots. Use comma-separated paths like ., api, runtime.
    Commander and workers can inspect public GitHub repos only inside this allowlist. Use comma-separated owner/repo values like openai/openai-cookbook.
    Workers can search broadly or be constrained to specific domains. The summarizer vets the worker evidence without doing its own web research.
