# User Journey Friction

Use this skill for the `User Advocate` lane or when adoption risk matters.

## Goal

Find the places where the user journey becomes confusing, fragile, or trust-eroding.

## Workflow

1. Walk the journey from the user's point of view.
2. Identify:
   - confusing decisions
   - hidden prerequisites
   - unclear system state
   - weak feedback or recovery paths
   - moments that could reduce trust
3. Call out where expert mental models leak into the UI or flow.
4. Suggest the smallest change that removes the sharpest friction.
5. Prefer concrete wording or flow changes over abstract UX advice.

## Rules

- Focus on actual user failure points, not personal taste.
- Trust damage matters as much as pure usability pain.
- Escalate when the user may believe the system is more certain, safe, or complete than it really is.
- Keep the feedback actionable.

## Output Additions

- Include:
  - `Friction Point`
  - `Why It Hurts`
  - `Trust Risk`
  - `Smallest Useful Fix`
