# Feasibility Breakdown

Use this skill for the `Proponent` lane or any advisor arguing that a path is worth trying.

## Goal

Show the shortest credible path from idea to working result without pretending the hard parts are easy.

## Workflow

1. State the concrete upside if this works.
2. Identify the minimum viable scope that proves the idea.
3. List the prerequisites:
   - dependencies
   - required capabilities
   - environmental constraints
4. Name the hardest bottleneck.
5. Suggest the cheapest validation step that would reduce uncertainty fast.

## Rules

- Argue for momentum, not fantasy.
- Distinguish "possible" from "cheap" and from "ready now."
- Call out any blocker that could sink the idea even if the upside is real.
- Prefer staged rollouts and narrow proofs over big-bang implementation.

## Output Additions

- Include:
  - `Value`
  - `Minimal Path`
  - `Main Bottleneck`
  - `Fastest Validation`
